# Handoff: RUNSHOES 목록 화면 (표본 전시실 컨셉)

## Overview
러닝화 정보·리뷰 아카이브 RUNSHOES의 목록(랜딩) 화면. 구매 버튼 없이 스펙과 리뷰만 다루는 정보 페이지. 데미언 허스트의 표본 전시(유리 상자, 두꺼운 흰 프레임) 시각 언어를 차용해 신발을 "계측 대상"으로 다룬다.

## About the Design Files
이 번들 안의 `RunShoes List.dc.html`은 **디자인 레퍼런스**로 만들어진 HTML 프로토타입입니다. 실제 프로덕션 코드로 그대로 복사해 쓰는 파일이 아니라, 의도한 룩앤필과 동작을 보여주는 참고 자료입니다. 목표는 이 디자인을 대상 코드베이스의 기존 환경(React, Vue, 네이티브 등)과 기존 패턴/라이브러리를 사용해 다시 구현하는 것입니다. 만약 아직 정해진 프레임워크가 없다면, 이 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

## Fidelity
**High-fidelity(hifi)** — 색상, 타이포그래피, 간격, 인터랙션이 확정된 상태의 목업입니다. 기존 코드베이스의 라이브러리/패턴을 사용해 픽셀 단위로 재현해주세요.

## Screens / Views

### 1. 목록 화면 (단일 스크롤 페이지)

**Purpose**: 러닝화를 용도별로 필터링/정렬해 탐색하고, 카드 클릭 시 상세 스펙을 모달로 확인.

**Layout (위→아래)**:
1. 헤더 — sticky, `padding: 20px 48px` (모바일 `16px 20px`), 배경 `oklch(0.965 0.002 90 / 0.92)` + `backdrop-filter: blur(8px)`, 하단 1px 보더 `oklch(0.8 0 0)`
2. 히어로 — 다크 배경 `oklch(0.11 0 0)`, 높이 `66vh`(min 460px, 모바일은 auto), `padding: 40px 48px`
3. 필터 바 — `padding: 32px 48px 0`
4. 정렬 + 토글 행 — `padding: 20px 48px 0`
5. 카드 그리드 — `padding: 28px 48px 100px`
6. 상세 모달 — 카드 클릭 시 전체 화면 오버레이

### 헤더
- 로고 "RUNSHOES" — 20px, weight 800, letter-spacing -0.02em
- 메뉴 3개: 목록(진하게, 15px/600) · 러너들의 선택(흐리게, 15px/500, `oklch(0.4 0 0)`) · 등록·관리(더 작고 흐리게, 13px/500, `oklch(0.62 0 0)`, 우측 끝)
- 모바일(~767px): 메뉴 숨기고 햄버거 버튼(줄 3개, 22×2px) → 클릭 시 드롭다운 메뉴

### 히어로 — 해부도 (3분할)
- 러닝화 **한 짝**이 3개 패널(`flex: 1` 각각, `gap: 1px`)에 걸쳐 하나의 이미지로 이어져 보이도록 구성 (패널 경계에서 잘려 보이되 연속된 형태 — 허스트의 3분할 수조 구조)
- 각 패널 배경 `oklch(0.13~0.14 0 0)`
- 스펙 라벨 3개, 신발의 실제 대응 부위 근처에서 짧은 선(1px, `oklch(0.55 0 0 / 0.6)`)이 뻗어 텍스트와 연결:
  - 드롭 10mm — 뒤꿈치·앞꿈치 잇는 위치 / "뒤꿈치와 앞꿈치의 높이 차"
  - 스택하이트 36mm — 미드솔 옆면 두께 방향 / "발밑에 깔린 쿠션의 양"
  - 미드솔 ReactX — 미드솔 중앙 하단 / "쿠션층에 쓰인 폼 소재"
  - 라벨명 14px/700 색상 `#1D7A72`(강조색), 설명 12px `oklch(0.6 0 0)`, `white-space: nowrap`로 한 줄 유지
- 하단 카피: 메인 "러닝화, 제대로 읽기" 52px/800/`oklch(0.97 0 0)`, 부제 "스펙은 빠짐없이, 용어는 설명과 함께" 17px/`oklch(0.62 0 0)`
- 모바일(~767px): 3패널을 세로 스택(`flex-direction: column`, 각 160px), 연결선 제거하고 이미지 아래 설명 3개 세로 나열, 카피 32px로 축소

### 필터 바
- 6개 칩(가로 스크롤, `overflow-x: auto`, 스크롤바 숨김): 전체 / 조깅·회복주 / 템포·인터벌 / LSD·장거리 / 레이스 / 트레일
- 칩: `padding: 6px 13px`, `border-radius: 100px`, 13px/600, 보더 `oklch(0.78 0 0)`
- 선택된 칩만 배경/보더 `#1D7A72` + 글자 흰색. 그 외엔 무채색

### 정렬 + 토글
- 정렬 3개 텍스트 버튼(평점순/리뷰순/좋아요순), 14px/700. 선택된 것만 색상 `#1D7A72`, 나머지 `oklch(0.35 0 0)`
- 우측 "찜한 것만 보기" 스위치(40×24px 트랙, 20px 노브). on 상태 트랙 배경 `#1D7A72`

### 카드 그리드
- 3열(1200px+) → 2열(768~1199px) → 1열(~767px)
- 그리드 전체에 상단/좌측 1px 보더, 각 카드는 우측/하단 1px 보더로 격자 구성 (카드 사이 갭 없음, 선으로만 구분)
- 카드 배경 `oklch(0.965 0.002 90)`, `padding: 18px`

**카드 내부 구성 (위→아래)**:
1. 이미지 영역 — `aspect-ratio: 4/3`, 흰 10px 보더(프레임) + `box-shadow: inset 0 0 0 1px oklch(0.85 0 0)`, 배경 `oklch(0.99 0 0)`
   - 카본 플레이트 배지(해당 시): 좌상단, 무채색 아웃라인 `border: 1px solid oklch(0.4 0 0)`, 배경 흰색, 글자 `oklch(0.3 0 0)` — **강조색 아님**
   - 찜 하트 버튼: 우상단, 32px 원형, 흰 반투명 배경. 찜 시 하트 색상 `#1D7A72`
   - 하단 얇은 기준선 1px `oklch(0.8 0 0)`
2. 표본번호·분류 — 모노스페이스(IBM Plex Mono) 11px, `oklch(0.55 0 0)`. 형식: `NO.01 · 조깅·회복주`
3. 모델명 — 19px/700 (가장 크게)
4. 브랜드명 — 12px/500, `oklch(0.58 0 0)` (모델명보다 작고 흐리게)
5. 한 줄 요약 — 상단 보더 구분, 13px `oklch(0.42 0 0)`
6. 스펙 3종(모노스페이스, 상단 보더 구분): 중량 Ng / 드롭 Nmm / 발볼 N — 라벨은 흐리게(`oklch(0.5 0 0)`), 값은 진하게(`oklch(0.2 0 0)`)
7. 최하단 행: 좌측 안정성(중립화/안정화, 12px/600), 우측 평점(18px/800, `#1D7A72`) + 리뷰 수(모노스페이스 11px)

### 상세 모달
- 카드 클릭 시 오픈. 배경 `rgba(10,10,10,0.6)` 오버레이, 중앙 정렬, `max-width: 760px`
- 흰 배경, `box-shadow: 0 20px 60px rgba(0,0,0,0.3)` (모달은 그림자 허용 예외)
- 좌: 이미지 영역(회색 배경) / 우: 표본번호·분류, 모델명(26px/800), 브랜드, 요약, 스펙 그리드(2열, 모노스페이스: 중량/드롭/스택하이트/발볼/안정성/미드솔), 평점+리뷰수
- 우상단 닫기 버튼(36px 원형)
- 모바일: `flex-wrap: wrap`으로 세로 스택, `max-height: 88vh` + `overflow-y: auto`로 스크롤 처리

## Interactions & Behavior
- 필터 칩 클릭 → 해당 category로 카드 목록 필터링 (client-side state)
- 정렬 버튼 클릭 → rating/reviews/likes 기준 정렬
- 찜 토글 스위치 → liked=true인 카드만 표시
- 카드 하트 클릭 → 개별 좋아요 상태 토글 (이벤트 버블링 막아 카드 클릭과 분리)
- 카드 클릭(하트 제외 영역) → 상세 모달 오픈
- 모달 배경 클릭 또는 닫기 버튼 → 모달 닫기 (모달 내부 클릭은 전파 차단)
- 모바일 햄버거 클릭 → 메뉴 드롭다운 토글
- 애니메이션: 토글 노브 `left` 전환 0.15s. 그 외 트랜지션 없음(정적 인터랙션 위주)

## State Management
- `category`: 선택된 필터 키 (기본 'all')
- `sort`: 정렬 키 (기본 'rating')
- `likedOnly`: boolean
- `liked`: { [shoeIndex]: boolean } 맵
- `selectedIdx`: 모달에 열린 신발 인덱스 (null이면 닫힘)
- `menuOpen`: 모바일 메뉴 토글

## Design Tokens

**Colors**
- 강조색(오직 하나): `#1D7A72` (딥 틸) — 선택된 필터, 찜 하트, 평점 숫자, 히어로 스펙 라벨에만 사용
- 다크 배경(히어로): `oklch(0.11 0 0)` ~ `oklch(0.14 0 0)`
- 라이트 배경(본문): `oklch(0.965 0.002 90)` (오프화이트, 갤러리 톤)
- 텍스트: `oklch(0.14~0.16 0 0)` (진함) ~ `oklch(0.55~0.65 0 0)` (흐림)
- 보더/구분선: `oklch(0.8~0.85 0 0)`
- 카드 프레임: 흰색 `#fff` + inset 보더 `oklch(0.85 0 0)`

**Typography**
- 본문: Noto Sans KR (400/500/600/700/800)
- 수치용 모노스페이스: IBM Plex Mono (400/500/600)
- 스케일: 히어로 메인 52px(모바일 32px) > 모달 모델명 26px > 카드 모델명 19px > 평점 18px > 라벨/버튼 13~14px > 스펙 수치/캡션 11~12px

**Spacing / Radius**
- 섹션 간 큰 여백(24~40px), 카드 내부는 촘촘(6~18px)
- 칩/토글: `border-radius: 100px` (완전 라운드), 카드/프레임: 각진 모서리(0 radius)
- 카드 그리드: 갭 없이 1px 보더로 격자 구성

**Shadows**
- 카드: 없음 (프레임/보더로만 구분)
- 모달만 예외적으로 `box-shadow: 0 20px 60px rgba(0,0,0,0.3)`

## Breakpoints
- 데스크톱: 1200px+ (3열 그리드)

- 모바일: ~767px (1열 그리드, 햄버거 메뉴, 필터 칩 가로 스크롤, 히어로 3분할→세로 스택)
