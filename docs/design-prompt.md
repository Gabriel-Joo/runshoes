# RUNSHOES — 디자인 프롬프트

러닝화 정보·리뷰 사이트 **RUNSHOES**의 목록 화면(랜딩)을 디자인해줘.

---

## 1. 컨셉

**"러닝화, 제대로 읽기"** — 초보 러너도 상급자도 같은 페이지를 보는 러닝화 정보 아카이브.

기존 러닝화 정보는 대부분 상급자 기준으로 쓰여 있어서, 드롭·스택하이트·프로네이션 같은 용어가 설명 없이 나열된다. RUNSHOES는 정보를 덜어내지 않는 대신 모든 용어에 즉시 확인 가능한 설명을 붙인다.

**쇼핑몰이 아니다.** 구매 버튼도, 가격 강조도, 세일 배지도 없다. 사용자의 목표는 "예쁜 걸 사는 것"이 아니라 **"나에게 맞는 신발을 찾는 것"** 이다.

---

## 2. 비주얼 컨셉 — 표본 전시실

데미언 허스트의 포름알데히드 표본 연작(유리 상자 안에 부유하는 표본, 두꺼운 흰 프레임, 갤러리 바닥)의 시각 언어를 차용한다.
러닝화를 **판매 상품이 아니라 관찰·계측의 대상인 표본**으로 다룬다.

- 카드는 얇은 헤어라인 테두리가 아니라 **두꺼운 흰 프레임**으로 감싼 유리 상자 형태. 신발 주변 여백을 넉넉히 준다
- 신발은 프레임 중앙에 **부유하듯** 배치한다. 바닥에 놓인 느낌은 피한다
- 배경은 순백이 아니라 **미묘한 회색 갤러리 톤**. 은은한 반사 허용
- 모든 신발 이미지는 **정측면 · 동일 방향(앞코가 왼쪽) · 동일 배율**로 통일한다. 표본함처럼 나란히 놓였을 때 규격이 맞아야 한다
- 카드 모서리는 각지게. 둥근 모서리는 쇼핑몰 문법이라 피한다
- 카드 사이는 여백이 아니라 **1px 선**으로 붙여 격자를 구성한다. 카드가 각각 떠 있는 게 아니라 전시장 격자 안에 들어가 있어야 한다

### 톤 레퍼런스
- 여백·타이포·절제: **On Running, Hoka 공식 사이트**
- 레이아웃 구조: **왓챠피디아** — 카드 그리드 + 평점 중심
- 스포츠 사이트에 흔한 역동적 이탤릭 대문자 헤드라인, 사선 분할, 속도감 그래픽은 쓰지 않는다

## 3. 화면 구성 (위 → 아래)

### ① 헤더
로고 `RUNSHOES` + 메뉴 3개

| 메뉴 | 라우트 |
|---|---|
| 목록 | `/` |
| 러너들의 선택 | `/best` |
| 등록·관리 | `/new`, `/edit/:id` |

`등록·관리`는 관리자 성격이므로 오른쪽 끝에 상대적으로 작고 흐리게 둔다.

### ② 히어로 — 해부도
- 다크 배경(near-black)
- **러닝화 한 짝을 3개 패널에 걸쳐 분할 배치** (허스트의 3분할 수조 구조)
- 각 패널에서 얇은 선이 뻗어나가 스펙 라벨과 연결된다

| 부위 | 라벨 | 설명 |
|---|---|---|
| 뒤꿈치·앞꿈치 | `드롭 8mm` | 뒤꿈치와 앞꿈치의 높이 차이입니다 |
| 미드솔 옆면 | `스택하이트 37mm` | 발밑에 깔린 쿠션의 양입니다 |
| 미드솔 | `미드솔 ReactX` | 쿠션층에 쓰인 폼 소재입니다 |

- 용어명은 강조색, 설명 문장은 회색. **말풍선 박스가 아니라 얇은 선 + 텍스트만**
- 카피 (왼쪽 정렬, 하단에 배치)
  - 메인: **러닝화, 제대로 읽기** — 48px 이상, 아주 크게
  - 부제: 스펙은 빠짐없이, 용어는 설명과 함께
- 화면 높이의 60~70%

### ③ 필터 바
용도별 6개 — `전체` `조깅·회복주` `템포·인터벌` `LSD·장거리` `레이스` `트레일`

칩 높이는 낮게. 시선 비중이 카드 그리드보다 커지면 안 된다.

### ④ 정렬 + 토글
- 정렬: `평점순` `리뷰순` `좋아요순`
- 우측: **"찜한 것만 보기"** 온·오프 스위치 (별도 페이지가 아니라 같은 화면의 토글)

### ⑤ 카드 그리드

카드 하나의 구성:

```
[ 신발 이미지 — 정측면, 부유, 아래 기준선 ]        ♡
NO.01 · 조깅·회복주            ← 표본번호 · 분류 (모노스페이스, 작게)
젤 카야노 32                   ← 모델명 (가장 크게)
아식스                         ← 브랜드 (작고 흐리게)
─────────────────────
발이 안쪽으로 무너지는 분에게    ← 한 줄 요약
─────────────────────
중량 300g   드롭 8mm   발볼 4E  ← 모노스페이스, 라벨은 흐리게
안정화                    4.5  리뷰 2
```

- 이미지는 카드에서 **가장 큰 자리**를 차지한다. 초보 러너에게 디자인은 실제 선택 기준이므로 작게 줄이지 않는다
- 이미지 하단에 얇은 **기준선**. 스택하이트 수치를 눈금처럼 표기. 정보가 없으면 `— —`
- 모든 **수치는 고정폭(모노스페이스) 폰트**. 실험 기록지 느낌
- `NO.01` 같은 표본번호 + 분류는 박물관 캡션 형식

## 4. 컬러

- **히어로는 다크(near-black), 그 아래는 라이트(회색빛 오프화이트)** — 스크롤하며 전환되는 리듬
- 무채색 기반. 강조색은 **딥 틸 `#1D7A72` 하나만**
- 강조색을 쓰는 곳: **선택된 필터 버튼 / 찜한 하트 / 평점 숫자 / 히어로 스펙 라벨**
- 그 외에는 유채색을 쓰지 않는다
- 틸은 채도가 낮아 흰 배경에서 약하다. **선택된 필터 버튼은 배경을 틸로 채우고 글씨를 흰색**으로 해 대비를 확보한다
- 카본 플레이트 배지는 강조색이 아니라 **무채색 아웃라인**으로 처리한다

---

## 5. 타이포

- 본문 산세리프 1종 + **수치용 모노스페이스 1종**
- **위계를 크게 벌린다** — 히어로 카피는 아주 크게, 모델명은 중간, 스펙 수치는 작게. 전부 비슷한 크기로 만들지 않는다
- 브랜드명은 모델명보다 작고 흐리게
- 사용하는 굵기는 2단계까지

## 6. 반응형 (필수)

브레이크포인트: **데스크톱 1200px+ / 태블릿 768~1199px / 모바일 ~767px**

| 요소 | 데스크톱 | 태블릿 | 모바일 |
|---|---|---|---|
| 카드 그리드 | 3열 | 2열 | 1열 |
| 헤더 메뉴 | 가로 나열 | 가로 나열 | 햄버거 또는 하단 고정 탭 |
| 필터 바 | 한 줄 | 한 줄 | 가로 스크롤 칩 |
| 히어로 3분할 | 가로 3패널 | 가로 3패널 | 세로 3패널 또는 단일 이미지 |
| 해부 연결선 | 표시 | 표시 | **제거** — 이미지 아래 설명 3개를 세로로 |

- 모바일에서도 **기능은 데스크톱과 100% 동일**하다. 크기와 배치만 바뀐다
- 목록 → 상세 모달 흐름은 모든 화면 크기에서 유지된다
- 모달은 모바일에서 폭·스크롤 처리를 반드시 한다

## 7. 하지 말 것

- **장식용 그라데이션** — 컬러풀한 배경 그라데이션, mesh gradient, 버튼 그라데이션. 단, 히어로 이미지 위 가독성용 어두운 오버레이는 허용
- **카드 드롭섀도우** — 프레임과 격자선으로 구분한다. 모달·드롭다운 같은 실제 오버레이에만 그림자 허용
- 이모지
- **모든 요소를 똑같은 회색 카드에 담기** — 위계가 사라진다
- **균일한 여백** — 섹션 사이는 크게 벌리고 카드 안은 촘촘하게. 리듬을 만든다
- 딥 틸 외의 유채색
- 구매 버튼, 가격 강조, 할인 배지 등 커머스 요소
- 신발 이미지의 각도·방향·배율이 서로 다른 것
